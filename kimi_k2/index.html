Build Process Timeline

A step-by-step chronology of the troubleshooting and build process

1

Initial FFmpeg Conflict

CRITICAL

The build process encountered its first major obstacle with FFmpeg library conflicts. openSUSE's dual FFmpeg system created package installation issues that prevented the build environment from being properly configured.

Problem: ffmpeg-mini vs ffmpeg-full package conflict
2

FFmpeg Resolution

FIXED

Explicit BuildRequires were added for the full ffmpeg-8 package from Packman, along with Conflicts statements to remove the mini version. This ensured all necessary development headers were available.

Solution: Force full ffmpeg installation, remove mini version
3

Missing Dependencies

DISCOVERY

CMake configuration revealed additional missing system libraries: OpenH264 for video encoding, libsrtp2 for secure RTP, and libcrc32c for checksum functionality. Each required careful identification and installation.

Dependencies: libopenh264-devel, libsrtp2-devel, libcrc32c-devel
4

Abseil Compatibility Crisis

CRITICAL

The most complex challenge involved Abseil library compatibility issues. The source code used various Abseil features without proper header includes, leading to cascading compilation errors that required systematic resolution.

Issues: absl::Nonnull, absl::Nullable, absl::optional, absl::string_view
5

Systematic Resolution

RESOLVED

Implemented sed-based replacements in the %prep section to convert Abseil types to C++17 standard equivalents. This "nuclear option" approach systematically addressed each compilation error as it appeared.

Method: sed replacements for type conversion
6

Final Build Success

COMPLETE

After resolving all dependency conflicts and compatibility issues, the build process completed successfully. The final package uses system libraries and maintains proper security update paths.

Result: Working libtg_owt package with system dependencies

Abseil Fix - Sed Commands

%prep section fixes 
# Fix absl::Nonnull and absl::Nullable
sed -i 's/absl::Nonnull<\([^>]*\)>/\1/g' \
  src/third_party/webrtc/rtc_base/pending_task_safety_flag.h

sed -i 's/absl::Nullable<\([^>]*\)>/\1/g' \
  src/third_party/webrtc/rtc_base/environment_factory.h

# Fix absl::optional
sed -i 's/absl::optional/std::optional/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h
sed -i 's/absl::nullopt_t/std::nullopt_t/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h
sed -i 's/absl::nullopt/std::nullopt/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h

# Fix absl::string_view
sed -i 's/absl::string_view/std::string_view/g' \
  src/third_party/webrtc/rtc_base/media_constants.h

Dependency Management

Complete list of dependencies and their roles in the build process

FFmpeg

Video/Audio Processing

Core multimedia library for WebRTC audio/video processing. Required full version with development headers.

Package: ffmpeg-8-devel
Source: Packman Repository

OpenH264

Video Codec

H.264 video encoding library required for WebRTC video transmission and processing capabilities.

Package: libopenh264-devel
Source: openSUSE OSS

libsrtp2

Secure RTP

Secure Real-time Transport Protocol library for encrypted WebRTC communication channels.

Package: libsrtp2-devel
Source: openSUSE OSS

libcrc32c

Checksum Library

CRC32C checksum calculation library for data integrity verification in WebRTC protocols.

Package: libcrc32c-devel
Source: openSUSE OSS

Abseil C++

Google Libraries

Google's C++ common libraries providing enhanced types and utilities. Required compatibility fixes.

Package: abseil-cpp-devel
Source: openSUSE OSS

CMake

Build System

Cross-platform build system generator that orchestrates the entire compilation and linking process.

Package: cmake
Source: openSUSE OSS
