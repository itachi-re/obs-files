Troubleshooting Guide

Comprehensive solutions for common build problems and errors

Common Issues

Build Fails Immediately

The build process fails right after starting with dependency conflicts.

Error Message

nothing provides ffmpeg-8-devel needed by libtg_owt

Solution

Enable the Packman repository and set proper priority:

Fix FFmpeg repository 
# Add Packman repository with high priority
sudo zypper ar -f -p 90 https://ftp.gwdg.de/pub/linux/misc/packman/suse/openSUSE_Tumbleweed/ packman
sudo zypper refresh

# Install FFmpeg development packages
sudo zypper install ffmpeg-8-devel

Package Conflicts

Conflicts between ffmpeg-mini and ffmpeg-full packages.

Conflict Message

file /usr/lib64/libavcodec.so.58 from install of ffmpeg-8-1:4.4-1.4.x86_64 conflicts with file from package ffmpeg-mini-1:4.4-1.4.x86_64

Solution

Add Conflicts statements to your spec file:

Add to spec file 
# Add these lines to your spec file header
Conflicts:      ffmpeg-8-mini-libs
Conflicts:      ffmpeg-8-mini-devel

FFmpeg Problems

Missing FFmpeg Development Headers

CMake cannot find FFmpeg headers during configuration.

CMake Error

Could NOT find FFmpeg (missing: FFmpeg_INCLUDE_DIRS)

Solution

Install the correct FFmpeg development package:

Install FFmpeg dev package 
# Remove conflicting packages first
sudo zypper remove ffmpeg-8-mini-devel ffmpeg-8-mini-libs

# Install full FFmpeg development package
sudo zypper install ffmpeg-8-devel

FFmpeg Version Mismatch

Different FFmpeg versions causing compatibility issues.

Version Check

Check FFmpeg versions 
# Check installed FFmpeg packages
rpm -qa | grep ffmpeg

# Check available versions
zypper se -s ffmpeg-8

Solution

Ensure consistent FFmpeg version across all packages:

Force consistent version 
# Install specific version if needed
sudo zypper install ffmpeg-8-devel-4.4.1

Abseil Compatibility Issues

absl::Nonnull Not Found

Compilation fails because absl::Nonnull is not recognized.

Compilation Error

error: 'absl::Nonnull' has not been declared

Solution

Add sed command to spec file %prep section:

Fix absl::Nonnull 
# In %prep section of spec file
sed -i 's/absl::Nonnull<\([^>]*\)>/\1/g' \
  src/third_party/webrtc/rtc_base/pending_task_safety_flag.h

absl::optional Not Found

Missing absl::optional type definitions during compilation.

Compilation Error

error: 'absl::optional' does not name a type

Solution

Replace with C++17 std::optional:

Fix absl::optional 
# In %prep section of spec file
sed -i 's/absl::optional/std::optional/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h
sed -i 's/absl::nullopt_t/std::nullopt_t/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h
sed -i 's/absl::nullopt/std::nullopt/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h

Patch Application Failed

Manual patch files fail to apply correctly.

Patch Error

1 out of 1 hunk FAILED -- saving rejects to file

Solution

Use sed commands instead of patches:

Use sed instead of patches 
# Instead of patches, use sed in %prep section
%prep
%setup -q

# Apply fixes with sed (more reliable than patches)
sed -i 's/absl::Nonnull<\([^>]*\)>/\1/g' src/file1.h
sed -i 's/absl::optional/std::optional/g' src/file2.h
# ... etc

CMake Configuration Errors

Missing Dependencies

CMake cannot find required system libraries.

CMake Error

Could NOT find OpenH264 (missing: OpenH264_LIBRARIES)

Solution

Install missing development packages:

Install missing dependencies 
# Install all required dependencies
sudo zypper install libopenh264-devel libsrtp2-devel libcrc32c-devel

Wrong CMake Flags

Incorrect CMake flags preventing system library usage.

Configuration Warning

Using bundled libraries because TG_OWT_PACKAGED_BUILD is not set

Solution

Set correct CMake flag in spec file:

Correct CMake configuration 
# In spec file %build section
%cmake . \
  -DCMAKE_BUILD_TYPE=Release \
  -DTG_OWT_PACKAGED_BUILD:BOOL=OFF \
  -DCMAKE_INSTALL_PREFIX=/usr \
  -DCMAKE_SKIP_RPATH=ON

Build Failures

Compilation Errors

Source code compilation fails with various errors.

Generic Error

error: expected ';' before '}' token

Debugging Steps

  1. 1. Check the exact error location in the build log
  2. 2. Examine the source file around the error
  3. 3. Look for missing includes or Abseil issues
  4. 4. Apply appropriate sed fixes
  5. 5. Test with a local build before OBS

Linking Errors

Linking phase fails with undefined symbols.

Linking Error

undefined reference to `avcodec_send_packet'

Solution

Ensure all libraries are properly linked:

Check library linking 
# Check if libraries are available
pkg-config --libs libavcodec libavformat libavutil

# Verify library files exist
ls -la /usr/lib64/libavcodec.so*

Runtime Issues

Missing Runtime Dependencies

Package installs but fails at runtime with missing libraries.

Runtime Error

error while loading shared libraries: libavcodec.so.58: cannot open shared object file

Solution

Add proper runtime requirements to spec file:

Add runtime requirements 
# Add to spec file
Requires:       ffmpeg-8-libavcodec
Requires:       ffmpeg-8-libavformat
Requires:       ffmpeg-8-libavutil

Performance Issues

Poor performance or unexpected behavior at runtime.

Performance Check

Check library versions 
# Check which libraries are being used
ldd /usr/lib64/libtg_owt.so | grep -E "(ffmpeg|openh264)"

# Check for any bundled libraries
strings /usr/lib64/libtg_owt.so | grep -i "bundled\|static\|embedded"

OBS Build Service Problems

Build Environment Issues

OBS build environment doesn't match local environment.

OBS Error

unresolvable: nothing provides ffmpeg-8-devel

Solution

Configure OBS project repositories:

Configure OBS repositories 
# Add Packman repository to OBS project
osc meta prj -e home:$(whoami)

# Add this repository section:

  
  
  x86_64

Local vs Remote Build Differences

Package builds locally but fails in OBS.

Debugging Steps

  1. 1. Check OBS build log for specific errors
  2. 2. Compare local and OBS build environments
  3. 3. Verify all dependencies are available in OBS
  4. 4. Check for any build-time network requirements
  5. 5. Test with osc build --clean flag
