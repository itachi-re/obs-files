Complete Build Guide

A comprehensive step-by-step guide to building libtg_owt for openSUSE Tumbleweed

Prerequisites

System Requirements

  • • openSUSE Tumbleweed (latest)
  • • x86_64 architecture
  • • 4GB+ available RAM
  • • 2GB+ disk space
  • • Internet connection

Required Tools

  • • osc (Open Build Service client)
  • • git
  • • cmake
  • • gcc-c++
  • • rpm-build

Important Note

Ensure you have proper permissions and access to the Open Build Service. Some packages require the Packman repository to be enabled for full FFmpeg support.

Environment Setup

1. Install Required Packages

Install build tools and dependencies 
# Install basic build tools
sudo zypper install osc git cmake gcc-c++ rpm-build

# Add Packman repository for full FFmpeg
sudo zypper ar -f https://ftp.gwdg.de/pub/linux/misc/packman/suse/openSUSE_Tumbleweed/ packman
sudo zypper refresh

# Install development dependencies
sudo zypper install ffmpeg-8-devel libopenh264-devel libsrtp2-devel libcrc32c-devel abseil-cpp-devel

2. Configure OBS Client

Setup Open Build Service client 
# Configure OSC (first time setup)
osc ls
# Enter your OBS username and password when prompted

# Create working directory
mkdir -p ~/obs/libtg_owt
cd ~/obs/libtg_owt

# Checkout your home project
osc checkout home:$(whoami)
cd home:$(whoami)

FFmpeg Repository Priority

The Packman repository must have higher priority than the main OSS repository to ensure the full FFmpeg package is installed instead of ffmpeg-mini.

Set Packman priority 
# Set Packman priority to 90 (lower number = higher priority)
sudo zypper mr -p 90 packman

# Verify repository priorities
zypper lr -p

Abseil Compatibility Fixes

The libtg_owt source code uses various Abseil library features without proper header includes. These sed commands in the %prep section resolve the compatibility issues.

1. Fix absl::Nonnull and absl::Nullable

Replace nullability annotations 
# Fix absl::Nonnull in pending_task_safety_flag.h
sed -i 's/absl::Nonnull<\([^>]*\)>/\1/g' \
  src/third_party/webrtc/rtc_base/pending_task_safety_flag.h

# Fix absl::Nullable in environment_factory.h
sed -i 's/absl::Nullable<\([^>]*\)>/\1/g' \
  src/third_party/webrtc/rtc_base/environment_factory.h

2. Fix absl::optional Types

Replace optional types with C++17 equivalents 
# Fix absl::optional in rtp_source.h
sed -i 's/absl::optional/std::optional/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h
sed -i 's/absl::nullopt_t/std::nullopt_t/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h
sed -i 's/absl::nullopt/std::nullopt/g' \
  src/third_party/webrtc/rtc_base/rtp_source.h

3. Fix absl::string_view

Replace string_view with C++17 equivalent 
# Fix absl::string_view in media_constants.h
sed -i 's/absl::string_view/std::string_view/g' \
  src/third_party/webrtc/rtc_base/media_constants.h

Why These Fixes Are Necessary

The libtg_owt source code was written assuming bundled Abseil libraries with specific include patterns. When using system-installed Abseil, these type definitions aren't automatically available, requiring explicit conversion to standard C++17 equivalents.

Build Process

1. Local Build Testing

Test build locally before OBS 
# Create package directory
osc mkpac libtg_owt
cd libtg_owt

# Add files to package
osc add libtg_owt.spec
osc add libtg_owt-0.1.0.tar.xz

# Test local build
osc build openSUSE_Tumbleweed x86_64 \
  --local-package \
  --clean

# Monitor build log
osc buildlog openSUSE_Tumbleweed x86_64

2. CMake Configuration

The key to using system libraries is the TG_OWT_PACKAGED_BUILD flag:

CMake configuration flags 
%cmake . \

  -DCMAKE_INSTALL_PREFIX=/usr \
  -DCMAKE_SKIP_RPATH=ON \
  -DBUILD_SHARED_LIBS=ON

Verification

1. Check Build Log

Monitor build progress 
# Check build log for errors
osc buildlog openSUSE_Tumbleweed x86_64 | tail -50

# Check for specific issues
grep -i "error\|failed\|missing" build.log

2. Test Installation

Install and test package 
# Install built package
sudo zypper in -f /var/tmp/build-root/home/abuild/rpmbuild/RPMS/x86_64/libtg_owt-*.rpm

# Verify installation
ldconfig -p | grep tg_owt
pkg-config --exists tg_owt && echo "Package found"

3. Dependency Verification

Check library dependencies 
# Check library dependencies
ldd /usr/lib64/libtg_owt.so | grep -E "(ffmpeg|openh264|srtp|crc32c)"

# Verify no bundled libraries
strings /usr/lib64/libtg_owt.so | grep -i "bundled\|static"

Success Indicators

  • • Build completes without errors
  • • All dependencies are dynamically linked
  • • Package installs successfully
  • • Library loads without missing symbols

3. Distribution

Once the package builds successfully in OBS, it will be automatically published to your home project repository and can be installed by users.

Install from OBS 
# Add your OBS repository

# Install package
sudo zypper refresh
sudo zypper install libtg_owt
